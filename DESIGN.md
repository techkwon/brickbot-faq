# Brickbot FAQ DESIGN.md

## 1. 목표 분위기
- 2026년 2권역 중등 강사진이 모바일에서 빠르게 답을 찾는 공개 FAQ다.
- 네오브루탈리즘의 명확함과 교육 문서의 안정감을 결합한다.
- 특정 브랜드를 복제하지 않고 Notion의 읽기 구조와 Linear의 필터 정보 구조만 참고한다.

## 2. 색상 역할
- 배경: `#F5F0E8` 따뜻한 종이색
- 카드: `#FFFDF8`
- 주요 텍스트/테두리: `#111111`
- CTA·링크: `#2F64FF`
- 핵심 강조: `#FFD84D`
- 보조 강조: `#FF7BC3`, `#7BE0C3`
- 주의: `#FF795E`
- 성공: `#59C36A`

## 3. 타이포그래피
- 외부 폰트를 호출하지 않는다.
- 기본: Pretendard가 있으면 사용하고 `Apple SD Gothic Neo`, `Noto Sans KR`, system-ui로 폴백한다.
- 큰 제목은 800~900, 본문은 400~600을 사용한다.
- 본문 줄높이는 1.65 이상, 한 줄 최대 폭은 72ch다.

## 4. 컴포넌트
- 카드: 3px 검정 테두리, 0~8px radius, `6px 6px 0 #111` 그림자.
- 버튼·필터: 2~3px 테두리, 누르면 그림자가 줄고 2px 이동한다.
- FAQ: `<details>` 기반으로 키보드 접근성과 무자바스크립트 기본 기능을 유지한다.
- 검색: 큰 입력창과 검색 결과 수를 함께 제공한다.
- 상태 배지: 색상만으로 의미를 전달하지 않고 텍스트를 병기한다.

## 5. 카피 톤
- 짧고 정확한 존댓말.
- 공개 페이지에 없는 내부 링크는 `단톡방 종합안내에서 확인해 주세요`로 안내한다.
- `공개용으로 정리된 내용입니다`라는 범위 안내를 명시한다.

## 6. Do / Don't
### Do
- 모바일 우선, 44px 이상 터치 영역, 명확한 포커스 링.
- 날짜·집계 기간·공개 여부를 항상 표시한다.
- 빈 날짜도 `새 FAQ 없음`으로 사실대로 표시한다.

### Don't
- 카카오톡 원문, 작성자, 메시지 ID, 내부 링크, 개인정보를 렌더링하지 않는다.
- 모든 카드에 무작위 색을 남발하지 않는다.
- 작은 회색 글씨나 색상만으로 상태를 구분하지 않는다.
- JavaScript 비밀번호나 숨겨진 내부 데이터 번들을 만들지 않는다.

## 7. 반응형
- 760px 이하에서는 단일 열.
- 검색·필터는 가로 스크롤 또는 줄바꿈.
- 본문 16px 미만 금지.

## 8. Agent Prompt Guide
- 이 파일을 먼저 읽고, 공개 데이터 정책과 접근성을 우선한다.
- 원문 데이터가 주어져도 UI 프로젝트로 복사하지 않는다.
- 완성 후 모바일·키보드·검색·빈 상태·404·보안 검사를 실행한다.
